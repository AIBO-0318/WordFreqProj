## 1. 개요
직접 크롤링한 **왓챠피디아 영화 코멘트(약 150만 건 / 309편)** 를 활용해
영화별 여론을 **단어 빈도 · 감성 · 유사 영화 추천**으로 보여 주는 Streamlit 웹 서비스.

핵심 차별점: 분석·학습·추천의 **모든 단계가 직접 수집한 데이터** 위에서 동작한다.
(공개 데이터셋 NSMC가 아니라 왓챠 코멘트로 감성모델을 학습하고, 추천도 코멘트에서 산출)

## 2. 문제 정의 및 목표
- 특정 영화에 대한 관객 반응을 **한눈에** 파악하기 어렵다(코멘트가 너무 많음).
- 목표:
  1. 영화/연도별 핵심 키워드 시각화
  2. 코멘트의 긍·부정 자동 분류
  3. "이 영화와 분위기·반응이 비슷한 영화" 추천

## 3. 시스템 구조 — 수집 → 준비 → 학습 → 서비스
```
[수집] 크롤러(왓챠/CGV/로저이버트/박스오피스)
   │        └ 왓챠 내부 API + Selenium, 세션쿠키·딜레이로 차단 회피
   ▼
[준비] preprocess.py (중복·길이·노이즈 제거)
   │   make_watcha_labeled.py (KNU 감성사전 자동 라벨링)
   ▼
[학습] train_sentiment_model.py (LSTM 감성)  /  build_recommender.py (TF-IDF+호평률)
   ▼
[서비스] WordFreqWebDashboard.py (Streamlit 3탭)
```

## 4. 데이터 수집 (crawler/)
| 크롤러 | 대상 | 방식 |
|---|---|---|
| watcha_comment_crawler | 왓챠피디아 코멘트(메인) | Selenium(URL수집) + 내부 API(requests) |
| cgv_review_crawler | CGV 실관람평 | Selenium |
| rogerebert_crawler | rogerebert.com 리뷰 | Selenium + BeautifulSoup |
| boxoffice_crawler | KOBIS 박스오피스 | OpenAPI |

- 왓챠 내부 API(`/api/contents/{code}/comments`)를 분석해 코멘트를 페이지네이션 수집.
- **차단 회피**: 요청 간 랜덤 딜레이, 버스트 후 선제 대기, 429 백오프, 세션쿠키 재사용.
- 로그인 정보·API 키는 코드에서 분리해 `.env`(환경변수)로 관리.

## 5. 데이터 준비
- **정제**: 중복 제거, 5자 미만·노이즈(이모지/특수문자만) 제거, 유니코드 정규화 → `watcha_comments_clean.csv`.
- **자동 라벨링**: 별점이 없어 KNU 한국어 감성사전으로 코멘트 극성 점수화
  → 점수 ≥ +2 긍정 / ≤ −3 부정, 클래스 균형 샘플링 → `watcha_labeled.csv`.

## 6. 감성분석 모델 (LSTM)
- 구조: `Embedding(32) → LSTM(64) → Dense(16, tanh) → Dense(2, softmax)`
- 전처리: Okt 토큰화 → 정수 인코딩(vocab 40,000) → 패딩(max_len=50)
- 학습: RMSprop, binary_crossentropy, EarlyStopping + ModelCheckpoint
- 추론 보정: 긴 코멘트의 과도한 부정 분류를 줄이려 부정 임계값(NEG_THRESHOLD=0.6) 적용.

## 7. 추천 모델 (콘텐츠 기반, 두 신호 결합)
크롤링 코멘트에서 **두 가지 신호**를 뽑아 결합한다.
- 🗣️ **분위기·소재**: 영화별 코멘트 명사를 TF-IDF 벡터화 → 코사인 유사도
- 😊 **관객 호불호**: LSTM 감성모델이 매긴 영화별 **호평률** 유사도
- 최종점수 `= α·분위기 + (1-α)·반응`, α는 대시보드 슬라이더로 조절.
- 결과 예: 기생충→설국열차·마더(봉준호), 인터스텔라→테넷·인셉션(놀란),
  어벤져스→마블 시리즈 — **코멘트 단어만으로 감독·시리즈 관계를 포착**.

## 8. 서비스 (Streamlit 대시보드)
1. **단어 빈도** — 워드클라우드·막대그래프·표/CSV 다운로드
2. **감성 분석** — 긍/부정 분포(도넛), 그룹별 특징어·대표 코멘트
3. **비슷한 영화 추천** — 추천 근거(공통 키워드)·분위기/반응/호평률 표시
- `@st.cache_data`/`@st.cache_resource`로 데이터·모델 캐싱.

## 9. 결과

## 10. 한계 및 회고
- **감성 라벨 품질이 병목**: 자동 라벨(감성사전)은 영화 주제어(상실·가난 등)를 부정으로
  오인 → 사람 기준 정확도가 평가치보다 낮음. 학습량을 늘려도 한계는 유지(라벨 한계).
  근본 해결책은 코멘트별 **실제 별점 라벨**이나 본 프로젝트에서는 수행하지 않음("실패" 지점).
- 추천의 호평률 분포가 73~99%로 좁아(모델 긍정 편향) 반응 신호는 보조적.
- 향후: 별점 기반 라벨/추천(협업필터링), CGV·로저이버트 데이터 활용한 한·영 비교.
